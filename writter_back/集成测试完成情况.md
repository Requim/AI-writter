### 3.2 API 层存根修复

#### 新建 `application/orchestrator.py`

```python
class NovelOrchestrator(AgentOrchestrator):
    """持有所有依赖，驱动 LangGraph 工作流"""
    - _ensure_workflow(): 懒初始化，PostgresSaver + compile
    - _make_config(thread_id): 构造 LangGraph configurable dict
    - invoke(): 启动新工作流
    - resume(): 恢复被中断的工作流
    - stream(): SSE 流式执行
```

#### `api/routers/novel_router.py`

| 端点 | 修复前 | 修复后 |
|------|--------|--------|
| `POST /novels` | 返回 `{"novel_id": "temp-id"}` | 创建 Novel 实体 → `repository.save()` → 返回真实 novel_id |
| `GET /novels` | 返回 `[]` | 调用 `repository.find_all_by_user()` |
| `GET /novels/{id}` | 返回硬编码示例 | 调用 `repository.find_by_id()` |
| `GET /novels/{id}/progress` | 返回假进度 | 调用 `repository.find_by_id()` → `novel.progress` |
| `GET /novels/{id}/chapters` | 返回 `[]` | 调用 `repository.find_by_id_with_chapters()` |

#### `api/routers/workflow_router.py`

| 端点 | 修复前 | 修复后 |
|------|--------|--------|
| `POST /workflows/{id}/invoke` | 返回硬编码 interrupt | 调用 `orchestrator.invoke()` 或 `orchestrator.resume()` |
| `GET /workflows/{id}/stream` | 返回 `{"status": "streaming"}` | 真实 SSE 流，yield chunk |

## 四、导入链修复记录

### 4.1 问题一：`repository.py` 相对导入层级错误

**错误**：`from ...service.entities.novel import Novel` — 超出顶层包

**修复**：改为绝对导入 `from service.entities.novel import Novel`

**涉及文件**：
- `writter_back/infrastructure/database/repository.py`
- `writter_back/infrastructure/memory/postgres_memory.py`

### 4.2 问题二：`pyproject.toml` 包配置错误

**错误**：`packages = ["domain", "infrastructure"]` — `domain` 目录不存在

**修复**：`packages = ["service", "infrastructure", "application", "api"]`

### 4.3 问题三：`PostgresSaver` 导入路径错误

**错误**：`from langgraph.checkpoint.postgres.aio import PostgresSaver`

**修复**：`from langgraph.checkpoint.postgres import PostgresSaver`

### 4.4 问题四：Windows 下 `psycopg2` 与 `langgraph-checkpoint-postgres` 不兼容

**错误**：`langgraph-checkpoint-postgres` 需要 `psycopg3`，但环境只有 `psycopg2-binary`

**修复**：`uv add "psycopg[binary]"` — psycopg3 自带 Windows 预编译 libpq

### 4.5 问题五：`PostgresSaver` 需要同步数据库 URL

**错误**：`PostgresSaver.from_conn_string("postgresql+asyncpg://...")` — asyncpg 格式不被接受

**修复**：在 `orchestrator.py` 新增 `_get_sync_db_url()` 方法，将 `+asyncpg` 去掉
